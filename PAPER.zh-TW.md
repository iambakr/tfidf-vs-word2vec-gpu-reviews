# TF-IDF vs. Word2Vec：技術性產品評論的情感分類比較研究

### 以 2,989 篇 Amazon NVIDIA RTX 40 系列顯示卡評論為例

**作者：** 洪凱迪（Kai-Tih Hong）· 國立臺北大學統計學系碩士 · 2025 年 7 月
**指導教授：** 黃怡婷 博士

> **TL;DR** — 在專業術語密集的 GPU 評論上，語義詞嵌入擊敗了傳統的詞頻統計特徵：在保留測試集上，最佳 Word2Vec 模型偵測出 **73.5%** 的負面評論，最佳 TF-IDF 模型僅為 **62.6%**（**相對提升 +17%**），且此結果經 300 組交叉驗證切分搭配 Friedman 檢定與 Dunn's 檢定驗證。同一套流程也可直接作為市場情報引擎：它自動識別出「電流聲（coil whine）」、到貨即損與退款摩擦是負面情緒的主要驅動因素。

---

## 執行摘要

高階顯示卡是價格昂貴、技術規格複雜的產品。多數消費者在購買前無法評估 VRAM 頻寬或散熱器噪音等規格，因此高度依賴線上評論——這是資訊不對稱的教科書案例。對製造商而言，這些評論則是一條尚未開發的即時產品回饋資料流。瓶頸在於規模：數以千計、非結構化且充滿術語的文本，沒有任何團隊能夠人工讀完。

本研究提出一個實務問題：**當文本充滿領域專業術語時，哪一種文本表示策略的情感分類效果更好——傳統的詞頻統計模型 TF-IDF，還是語義詞嵌入模型 Word2Vec？**

以 2,989 篇美國 Amazon 平台的 NVIDIA RTX 40 系列顯示卡評論為資料，本研究對 **52 組模型組態**進行了基準測試：兩大特徵工程家族（TF-IDF 搭配互資訊或 mRMR 特徵選取、共 5 種維度 × Word2Vec 共 3 種維度），交叉搭配四種分類器（SVM Linear、SVM RBF、Random Forest、XGBoost）。由於漏掉一則負面評論對製造商的代價遠高於誤讀一則正面評論，**負面類別的召回率（Recall）為主要指標**，並以 F1 分數作為平衡性的檢核。

**關鍵發現：**

1. **Word2Vec 明確勝出。** 統計上效能最佳的模型群組（依據 300 組交叉驗證切分上的 Dunn's 事後檢定）以 Word2Vec 組態為主：召回率前 12 名模型中有 9 組、F1 前 8 名模型中有 7 組是 Word2Vec。
2. **「Word2Vec + XGBoost」是最佳的負面評論偵測器**（測試集召回率 0.735）；**「Word2Vec + SVM RBF」則提供最佳的精確率／召回率平衡**（F1 最高達 0.694）。
3. **若必須使用 TF-IDF，請在低維度下搭配 mRMR。** 經 mRMR 篩選的 258 維特徵達到 0.626 的召回率——但效能隨維度增加而*衰退*（1,292 維時降至 0.361）。真正的限制因素是特徵冗餘，而非特徵不足。
4. **同一套流程可免費產出產品情報。** Bigram 網路分析與詞向量聚類揭露了具體、可行動的抱怨主題——硬體噪音（「電流聲（coil whine）」，以 170 次成為出現頻率最高的 bigram）、物流品質問題（「到貨即損（dead on arrival）」）以及購後摩擦（退款／RMA／客服）。

此方法可推廣至任何技術性產品類別的評論語料——GPU、筆電、相機、家電——並為建立自動化的客戶之聲（voice-of-customer）監測系統提供範本。

## 1. 為什麼 GPU 評論值得分析

GPU 市場變化迅速：新架構、AI 驅動的需求，以及高昂的價格（RTX 40 系列價格帶約從 300 美元到 2,000 美元）。三項特性使其評論成為 NLP 方法的理想壓力測試——同時也是有價值的商業資產：

- **高度資訊不對稱。** 消費者在購買前無法測試散熱、噪音或驅動程式穩定性，因此評論情感直接影響轉換率。一項 1,600 美元的產品，其負面評論的代價不成比例地高。
- **術語密集。** 評論中充滿低頻技術詞彙——*coil whine、DLSS、VRAM、form factor、ray tracing*——正是詞頻統計方法表現吃力、而語義詞嵌入理應發揮優勢的詞彙類型。
- **可行動性。** 抱怨可對應到具體的工程與營運槓桿：噪音（coil whine）、包裝／品管（到貨即損）、尺寸（機殼空間）以及售後服務（RMA）。

如果一個方法在這裡行得通，在更容易的語料上也會行得通。

## 2. 資料

**來源：** 美國 Amazon 平台的 NVIDIA RTX 40 系列顯示卡（ASUS、GIGABYTE、MSI）產品評論，以網路爬蟲蒐集。

| 屬性 | 數值 |
|---|---|
| 最終樣本數（預處理後） | **2,989 篇評論** |
| 訓練／測試分割 | 2,092 / 897（分層抽樣，70/30） |
| 標籤定義 | 1–3 星 → **BAD**，4–5 星 → **GOOD** |
| 類別比例（GOOD : BAD） | ≈ 4.76 : 1（2,470 vs. 519）——高度不平衡 |
| 訓練語料規模 | 63,309 個詞語（tokens），6,532 個不重複詞彙 |
| 品牌分佈 | GIGABYTE 40.1%、ASUS 33.5%、MSI 26.5% |
| 晶片分佈 | 涵蓋完整 RTX 40 系列；前三名：4060（19.4%）、4090（16.5%）、4070（16.0%） |

評論同時包含元數據——品牌、晶片與 VRAM（one-hot 編碼）、評論長度、有幫助（Helpful）票數、圖片／影片數量，以及顯示卡物理體積（長×寬×高，作為散熱設計的代理變數）——這些元數據與文本特徵合併為混合特徵集，用於每一組模型組態。4.76:1 的類別不平衡並非需要排除的干擾——它*就是*商業問題本身：負面評論很稀少，而稀少事件正是值得偵測的對象。

## 3. 研究方法

### 3.1 分析流程總覽

```
原始評論
  → 預處理（清理 → 斷詞 → 停用詞 → 詞形還原 → 領域詞彙校正）
  → 文本表示
      ├── 路徑 A：TF-IDF（2,748 個詞項）→ 特徵選取（MI 或 mRMR）
      │            → 258 / 517 / 776 / 1,034 / 1,292 維   （10 組特徵集）
      └── 路徑 B：Word2Vec（Skip-gram）→ 詞向量平均
                   → 100 / 200 / 300 維                    （3 組特徵集）
  → + 元數據特徵
  → 訓練 4 種分類器 × 13 組特徵集 = 52 組模型組態
  → 評估：召回率優先、F1 為輔 · 30 次重複 10 摺交叉驗證 · Friedman + Dunn's 檢定
```

### 3.2 文本預處理

標準的 NLP 清理流程（R：`quanteda`、`stopwords`、`textstem`），其中兩項領域專屬的加工值得特別說明：

1. **自定義停用詞** — 將不帶情感訊號的通用產品詞彙（*card、rtx、gpu、nvidia*）加入 snowball／smart 通用停用詞辭典。
2. **領域詞彙校正** — 將縮寫還原為領域內的常見形式（*fp → fps*、*temp → temperature*、*trace → tracing*），彌補通用 NLP 工具對硬體圈用語的盲點。

選擇詞形還原（Lemmatization）而非詞幹提取（Stemming），以保留詞彙語義（這對 Word2Vec 至關重要）。有 24 篇評論在過濾後成為空白而被移除。

### 3.3 兩種相互競爭的文本表示法

| | **TF-IDF** | **Word2Vec（Skip-gram）** |
|---|---|---|
| 核心概念 | 一個詞若在本篇文件中頻繁出現、卻在其他文件中罕見，則它很重要 | 一個詞的意義由它周圍的詞所定義 |
| 輸出 | 稀疏向量，每個詞彙對應一個維度 | 稠密向量（100–300 維），編碼語義鄰近性 |
| 能理解 *"terrible"* ≈ *"awful"* 嗎？ | 不能——不同字串就是不同維度 | 能——兩者的向量彼此鄰近 |
| 能處理罕見術語（*coil whine、DLSS*）嗎？ | 弱——罕見詞項的權重充滿雜訊 | 好——Skip-gram 對低頻詞的捕捉能力明確較強 |
| 維度問題 | 2,748 個稀疏維度 → 需要特徵選取 | 內建解決：維度本身就是訓練參數 |

**TF-IDF 路徑。** IDF 權重僅以訓練集擬合（嚴格的 fit-transform，無資料洩漏）。由於原始矩陣是高維且稀疏的，本研究比較了兩種特徵選取演算法，各自保留排名前 10/20/30/40/50% 的詞項（258–1,292 維）：

- **互資訊（Mutual Information, MI）：** 依「該詞項的出現能降低多少標籤的不確定性」為詞項排序。簡單，但忽略了所選詞項之間的冗餘性。
- **mRMR（minimum Redundancy Maximum Relevance，最小冗餘最大相關性）：** 在最大化與標籤的相關性的同時，*懲罰*與已選詞項之間的相關性——產出更精簡、共線性更低的特徵集。

**Word2Vec 路徑。** 以 Skip-gram（窗口 7、負採樣 7、45 次迭代）分別訓練 100、200、300 維的詞向量；每篇評論以其所有詞向量的平均值表示。

### 3.4 分類器與評估

四種分類器涵蓋主要的模型家族：**SVM（Linear）**、**SVM（RBF）**、**Random Forest**、**XGBoost**。超參數以隨機搜尋調校（每個模型抽樣 30 組參數組合，以交叉驗證平均召回率為主要挑選指標、F1 為並列時的參考依據）。類別不平衡在評估設計層面處理：約 4.76:1 的 GOOD:BAD 比例透過分層抽樣在訓練／測試分割與每一個交叉驗證摺中維持一致，且模型以召回率優先的方式挑選，使稀少的負面類別主導最佳化方向。SMOTE 等重抽樣技術則刻意留作未來研究（見 §7）。

**召回率優先的評估。** 在本應用情境中，偽陰性（漏掉一則真實抱怨）的代價遠高於偽陽性（把滿意的顧客誤標為待審查）。因此模型主要以 **BAD 類別的召回率**排序，並以 **F1 分數**作為次要的、著重平衡性的準則。

**統計嚴謹性。** 單一測試集的數字可能高估運氣好的模型。全部 52 組模型組態均在 **300 組完全相同的交叉驗證切分（30 次重複 10 摺）**上重新評估，先以 **Friedman 全局檢定**比較，達顯著後再以基準模型為參照執行成對的 **Dunn's 事後檢定（Holm 校正）**。與最佳模型無統計上顯著差異的模型，共同構成「效能最佳模型群組」。

## 4. 實證結果

### 4.1 核心比較（保留測試集）

| 架構 | 最佳組態 | 召回率（BAD） | 備註 |
|---|---|---|---|
| **Word2Vec** | 300 維 + **XGBoost** | **0.735** | 全部 52 組模型中最佳 |
| TF-IDF + mRMR | 258 維 + SVM Linear | 0.626 | TF-IDF 陣營最佳結果 |
| TF-IDF + MI | 258 維 + XGBoost | 0.619 | 跨維度表現穩定（0.594–0.619） |

「Word2Vec + XGBoost」偵測出 **73.5% 的負面評論**，較最佳 TF-IDF 組態**相對提升 +17.4%**。在 F1 分數方面，「Word2Vec + SVM RBF」以 **0.694**（300 維）領先。

![負面評論召回率：Word2Vec vs. TF-IDF](./results/performance_comparison.png)

### 4.2 差異具有統計上的真實性

Friedman 檢定壓倒性地拒絕了 52 組模型效能相等的假設（召回率：χ²(51) = 11,827，*p* < 0.001；F1：χ²(51) = 9,811，*p* < 0.001）。Dunn's 事後檢定進一步識別出與最佳模型在統計上無顯著差異的模型：

- **召回率最佳群組（12 組模型）：** 基準模型為「TF-IDF mRMR 517 維 + SVM RBF」（中位數 0.757）。**12 組中有 9 組是 Word2Vec 組態**；僅存的 TF-IDF 成員全部是低維度的 mRMR 特徵集（258/517 維）。
- **F1 最佳群組（8 組模型）：** 基準模型為「Word2Vec 100 維 + SVM RBF」（中位數 0.684）。**8 組中有 7 組是 Word2Vec 組態**；前七名全部為 Word2Vec + SVM／Random Forest。

模式相當一致：語義詞嵌入為*每一種*分類器家族提供了穩定且優質的特徵基礎，而 TF-IDF 只有在經過積極的冗餘控制式精簡後才保有競爭力。

### 4.3 三項深層發現

**1. mRMR 呈現「維度遞減效應」。** 使用 mRMR 特徵時，SVM Linear 的召回率從 0.626（258 維）*下滑*至 0.361（1,292 維）。更多特徵反而稀釋了訊號——這是對「特徵越多越好」直覺的一記警鐘，也證明了冗餘控制正是 TF-IDF 能夠保持可用的關鍵。值得注意的是，此衰退對 SVM 的衝擊最大，而樹狀集成模型幾乎持平——其內建的特徵子抽樣機制吸收了會癱瘓邊界型模型的冗餘。

![mRMR 維度遞減效應（依分類器）](./results/dimensionality_decay.png)

**2. 分類器的選擇應跟隨商業目標。** XGBoost 逐步修正殘差的增強機制使其對稀少的負面類別最為敏感（召回率最佳）；SVM RBF 則能利用詞嵌入空間的非線性結構，取得最佳的精確率（Precision）／召回率平衡（F1 最佳）。一條流程，兩種部署方案。

![召回率 vs. F1：所有 Word2Vec 組態全面領先](./results/recall_f1_tradeoff.png)

**3. Random Forest 在稠密詞嵌入上過度配適。** 訓練集召回率近乎完美（~1.0），到了測試集卻崩落至 0.665–0.690——是所有分類器中最大的泛化落差。在 TF-IDF 實驗中，XGBoost 的落差則是最小的。

## 5. 從分類到市場情報

一個只會輸出標籤的情感分類器只是半個產品。同一套流程還能回答顧客*為什麼*不滿意。

### 5.1 語料庫透露的訊息

整個語料庫中出現頻率最高的 bigram——領先所有正面詞組——是**「coil whine（電流聲，170 次）」**：一種高負載下的電流雜音，也是一個具體的噪音／品管工程改善目標。正面用語集中於 *work great、highly recommend、run cool、easy install*；效能情境詞組（*max setting、ultra setting、frame rate*）則顯示買家會立即對顯示卡進行極限測試。

Bigram 網路分析揭露了一個具有實務意涵的不對稱性：

- **正面評論在語言上高度集中** — 以單一樞紐（"great"）為核心，向效能、散熱與噪音等衛星節點放射。容易建模，少有意外。
- **負面評論則碎裂為多個彼此獨立的問題群集** — 硬體瑕疵（coil whine）、售後摩擦（*customer service、return*）、供電問題（*power supply、cable*）與驅動程式問題（*driver、update*）。每一個群集對應到公司內部不同的負責單位：品管、客服、配件、軟體。

### 5.2 經模型驗證的監測字典

對頂尖 TF-IDF 模型（mRMR 258 維 + SVM Linear / SVM RBF；論文圖 6.2.1）的特徵重要性分析，量化了真正驅動分類的特徵：正面方為 *great、quiet、love*；**return、annoy、terrible、refund** 則是消費者在抱怨時的共通語言。值得注意的是，有兩個非文本特徵也進入前 15 名——**`Tokens`**（評論長度）與 **`chipset.4070_Ti`**（論文將其標記為產品層級的「熱點」詞彙）——直接證明了文本 + 元數據的混合特徵確實承載真實訊號。

![前 15 名特徵重要性，mRMR 258 + SVM Linear / RBF](./results/feature_importance_top15.png)

將 Word2Vec 詞向量以主成分分析（PCA）投影至二維後，可見語義連貫的群集——「產品故障與退貨」區域（*faulty、return、refund*）、「遊戲效能與規格」區域（*cyberpunk、vram、dlss、core、memory*），以及「物理尺寸與安裝」區域（*form、factor、size、small、large*）。

**實務應用：** 這些經驗證的關鍵字集可建立為監測字典，即刻部署於 CRM 工單、社群論壇與社交管道，進行即時分流與議題路由——完全不需要重新訓練模型。

### 5.3 建議的生產環境架構

對於想建立自動化評論分析系統的 GPU 製造商或零售商：

1. **偵測層：** Word2Vec（300 維）+ XGBoost，以召回率為調校目標——盡可能捕捉所有負面訊號。
2. **平衡層：** Word2Vec + SVM RBF，用於精確率同等重要的儀表板級情感指標。
3. **解釋層：** bigram 網路 + 詞嵌入群集，定期更新，將被標記的評論轉化為已排序、已指派負責單位的議題主題（噪音 → 硬體品管；RMA 摩擦 → 客服營運；機殼相容性抱怨 → 產品設計／行銷文案）。

## 6. 將此方法套用至其他產業

流程中沒有任何環節是 GPU 專屬的。若要移轉到其他領域（筆電、家電、保養品、SaaS 評論）：

1. **務實地重新定義標籤。** 星等 → 二元標籤；依你的業務對「不滿意」的界線，決定 3 星這個中間地帶的歸屬。
2. **優先投資領域詞彙。** 自定義停用詞表與縮寫校正對照表是本研究中槓桿效益最高的預處理步驟；每個領域都有對應項（例如對筆電而言，*battery* 就不該是停用詞）。
3. **偵測任務預設採用詞嵌入 + 梯度提升**；保留低維度 mRMR TF-IDF 作為可解釋／輕量的備援方案——它幾秒鐘就能訓練完成，且其特徵是人類可讀的單詞。
4. **依你的經濟邏輯決定模型排序指標。** 當漏掉抱怨的代價高昂時，召回率優先是對的；當分析師審查標記的時間是瓶頸時，則以 F1 優先。
5. **在宣稱某個方法「勝出」之前，務必以重複交叉驗證 + 無母數檢定驗證** — 單次分割的排行榜經常產生誤導。

## 7. 研究限制與未來方向

**研究限制。** Amazon API 的技術限制使單次擷取最多只能取得約 100 則評論，對熱門產品構成抽樣偏誤（sampling bias）；研究結果來自單一平台、單一產品線，且僅限英文文本。研究發現應被視為一份嚴謹的個案研究，而非普遍適用的方法排名。

**為什麼採用傳統方法（而非 BERT）？** 本基準測試刻意將範圍限定於 transformer 之前的方法：TF-IDF 的特徵是人類可讀的單詞（在法規遵循與利害關係人溝通上是一項優勢），且兩種特徵家族的訓練成本都只是 transformer 的一小部分。如論文所述（引用 Wang & Manning, 2012），經過良好調校的線性模型搭配詞袋／TF-IDF 特徵，在某些資料集上的表現甚至不輸於複雜得多的模型——因此在動用更重的架構之前，它們正是應該先被超越的基準線。已知的代價（論文中亦有承認）是這類表示法忽略語序與否定詞（例如 "not bad"「不差」這類帶有否定詞的複雜情感）。

**部署上的注意事項。** 有一項元數據特徵——`Helpful`（社群投票數）——是在評論發佈*之後*才逐步累積的；新爬取的評論此特徵一律從 0 開始。生產系統應預期此特徵在推論階段貢獻的訊號會低於訓練階段。

**未來方向。**

- **閾值調整：** 透過精確率–召回率曲線的引導，將決策閾值降至 0.5 以下，可在漏掉抱怨代價高昂的情境中換取更高的召回率。
- **類別不平衡技術：** 對少數類別採用 SMOTE 式的合成過採樣。
- **Transformer 模型：** BERT 家族的編碼器可加入靜態詞嵌入無法捕捉的上下文理解能力（語序、否定詞、反諷）——是自然的下一個基準測試對象。

## 附錄 A — 中文摘要（Abstract in Traditional Chinese）

圖形處理器（GPU）市場因技術變革與新興需求而快速演進，消費者選購規格複雜的高單價顯卡時面臨顯著的資訊不對稱，決策高度依賴線上評論。本研究以美國 Amazon 平台 NVIDIA RTX 40 系列顯示卡評論（N = 2,989）為資料，比較傳統詞頻統計模型（TF-IDF，搭配互資訊與 mRMR 特徵選取）與詞嵌入技術（Word2Vec）在情感分類任務上的效能，共測試 52 組模型組合，並以 30 次重複 10 摺交叉驗證搭配 Friedman 與 Dunn's 檢定驗證顯著性。實證結果顯示，能捕捉語義關係的 Word2Vec 顯著優於 TF-IDF：「Word2Vec + XGBoost」在偵測負面評論上召回率最高（0.735），「Word2Vec + SVM RBF」則於 F1 分數表現最佳；TF-IDF 框架下 mRMR 於低維度展現較高效率。本研究亦透過詞組網路與詞向量視覺化，自動識別出「電流聲（coil whine）」、「到貨即損（DOA）」與退貨／維修等負面評論核心主題，為製造商提供一套數據驅動的市場意見分析方法。
